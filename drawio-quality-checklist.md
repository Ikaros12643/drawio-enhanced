# Draw.io 故障排查清单

这是故障排查资料，不是默认交付门禁。只有在图表打不开、渲染异常、导出失败、节点明显重叠、文本严重溢出，或用户明确要求精修时才读取。

默认交付前只做轻量检查：
- XML 格式合法
- ID 唯一
- 边的 source/target 存在
- 边包含 mxGeometry
- 节点没有明显重叠

不要为了轻微间距、边路由、美观微调或文本宽度估算反复重算坐标。

---

## 布局检查

### 1. 无节点重叠

**检查**：每对节点的边界框不能相交。

```
对于任意两个节点 A 和 B：
|Ax - Bx| >= max(Aw, Bw)  OR  |Ay - By| >= max(Ah, Bh)
```

**失败修复**：调整节点位置或减小 `NODE_W`/增加 `GAP_H`。

### 2. 画布足够大

**检查**：没有节点超出画布边界。

```
max(x + w) + CANVAS_PAD <= CANVAS_W
max(y + h) + CANVAS_PAD <= CANVAS_H
```

**失败修复**：扩大画布或移动节点。画布可以随内容自然扩展，不需要压缩到固定 X/Y 范围。

### 3. 间距一致

**检查**：相邻节点之间的间距应匹配 `GAP_H` / `GAP_V`（±10px 容差）。

**失败修复**：只调整明显不一致的节点。不要为了轻微间距误差重算全图坐标。

### 4. 对齐正确

**检查**：同一行的节点共享相同的 `y` 值。同一列的节点共享相同的 `x` 值。

**失败修复**：对齐坐标，确保同行/列节点坐标一致。

---

## 文本检查

### 5. 文字适配节点

**检查**：文字不能溢出节点边界。

估算：
- 中文字符 ≈ 14px/字 (字号 13)
- 英文字符 ≈ 8px/字 (字号 13)
- 文字宽度 > 节点宽度 - 20px (内边距) 时需处理

**失败修复**：
- 加宽节点
- 减小字号
- 添加 `<br>` 换行

### 6. 最小可读性

**检查**：
- 字号不低于 9px
- 节点不小于 80×30

**失败修复**：调整字号或节点尺寸。

### 7. 菱形文字

**检查**：菱形/菱形决策节点文字应简短（2-3 个短词），字号 11 而非 13。

**失败修复**：缩短标签或使用较小字号。

---

## XML 检查

### 8. 箭头目标存在

**检查**：每个 `source="X"` 和 `target="Y"` 必须对应一个已定义的 `id="X"` / `id="Y"` 节点。

**失败修复**：检查 ID 拼写，确保所有引用的节点都已定义。

### 9. ID 唯一

**检查**：整个 XML 中每个 `id` 属性必须唯一。

**失败修复**：使用递增 ID 或唯一前缀。

### 10. 正确的 parent

**检查**：所有内容元素必须有 `parent="1"`（除非有意分组到容器中）。

**失败修复**：添加或修正 `parent` 属性。

### 11. XML 格式正确

**检查**：
- 格式良好的 XML
- 所有属性用引号包裹
- HTML 实体已转义 (`&lt;` `&gt;` `&amp;` `&quot;`)
- **无 XML 注释** (`<!-- -->`)

**失败修复**：修正 XML 语法，转义特殊字符。

### 12. 边有 mxGeometry

**检查**：每个边的 `mxCell` 必须包含 `<mxGeometry relative="1" as="geometry" />` 子元素。自闭合边单元格无效。

**失败修复**：使用展开形式：
```xml
<mxCell id="e1" edge="1" parent="1" source="a" target="b" style="...">
  <mxGeometry relative="1" as="geometry" />
</mxCell>
```

## 样式检查

### 13. 主题一致性

**检查**：所有填充/描边/字体颜色来自选定主题。无临时十六进制值。

**失败修复**：将颜色替换为主题定义的颜色。

### 14. 箭头路由

**检查**：
- 箭头仅使用正交路由（水平 + 垂直）
- 无对角箭头
- 返回箭头沿外侧路由，不穿过无关节点

**失败修复**：添加 `edgeStyle=orthogonalEdgeStyle;` 到箭头样式。

### 15. 颜色预算

**检查**：遵循颜色预算规则（见 drawio-themes.md）：
- 3-5 节点：2-3 色
- 6-8 节点：3-4 色
- 9+ 节点：4-5 色
- 不超过 3 个相邻节点同色

**失败修复**：重新分配颜色角色，减少颜色数量。

### 16. html=1 标志

**检查**：任何包含 HTML 标签（`<b>`、`<br>`、`<font>` 等）的单元格样式必须包含 `html=1`。

**失败修复**：在样式字符串中添加 `html=1`。

---

## 快速检查脚本

生成 XML 后，按以下顺序快速检查：

```
1. XML 格式是否正确？ (无注释、实体已转义、属性有引号)
2. 所有 ID 是否唯一？
3. 所有箭头目标是否存在？
4. 是否有节点重叠？
5. 文字是否适配节点？
6. 颜色是否来自主题？
7. 箭头是否正交？
```

这些检查用于定位问题。优先修复会导致文件不可打开或明显不可读的问题；视觉微调最多尝试一次。
